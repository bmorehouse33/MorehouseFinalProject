Project Proposal

1.Project Description


For my final project, I have decided to create an application that allows users to check out study rooms based on what date they need the room, how long they will need a study room, and how many study rooms are open. Users will open the browser and enter the above information and the system will search for which rooms are available for the time selected. If the criteria for an open room meets the users request, then the user will be able to reserve the study room. If the user inputs a request that does not meet the criteria for an open room, the system will explain the error and have them try inputting a different date, time, and/or location to check out a study room. Because it will be very hard to input every single building and every single study room in that building on a campus, I will be populating the studyroom building locations and room numbers myself to simulate how the system would work.

2.Project Design


I plan to use an SQL database to store the users information as well as the study rooms that are available to check out. The use cases for my project are allowing users to enter the date, duration, and location to request the study room, allowing users to then reserve the study room, and allowing users to cancel, move, or change their study room request.
